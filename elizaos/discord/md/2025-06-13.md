# elizaOS Discord - 2025-06-13

## Overall Discussion Highlights

### ElizaOS Technical Updates
- **ElizaOS V2 Release**: Version 1.0.8 is out of beta with significant upgrades making development easier
- **Migration Progress**: Tools being developed to port plugins from V1 to V2 using LLMs
- **Version Update**: v1.0.9 was announced, addressing several bugs from previous versions
- **Database Issues**: Users encountered migration errors with the message "Failed query: CREATE SCHEMA IF NOT EXISTS $user"
- **Plugin Ecosystem**: The community was reminded that ElizaOS has over 180 plugins available

### Spartan Development
- Spartan is a separate stack from ElizaOS with different architecture focused on DeFi and trading tools
- First product release is imminent, though specific features will be simple initially
- Development will be reactive to community feedback
- Documentation will be created after launch rather than before

### Auto.fun Progress
- Currently undergoing redesign based on feedback from launch partners

### Communication Challenges
- Twitter/X account suspension remains a significant concern for the community
- Team is working with X to restore accounts while expanding communication across multiple platforms
- Jin mentioned plans to fix news pipelines to improve information flow
- Community suggested creating an official links channel to consolidate resources during social media disruptions

### Business Questions
- Community asked about profitability; Odilitime indicated they're not yet profitable but have sufficient runway
- Focus remains on product development and release rather than immediate profitability

## Key Questions & Answers

**Q: Do we have any useful or utility agent other than shit-posting?**  
A: "We have over 180 plugins that make it do more than shit posting. You add things like plugin-mcp and the sky is the limit." (Odilitime)

**Q: Do we have any significant updates for Spartan along with v2 release?**  
A: "V2 is out of beta, lots of developers starting to use it. We're already up to 1.0.8 with all the feedback we've gotten. Spartan is about to have his first product release, stay tuned." (Odilitime)

**Q: Will First product release happen along with v2 announcement?**  
A: "It's possible but we're not attempting to do that, they are separate things." (Odilitime)

**Q: Is ai16z profitable? And making good money?**  
A: "I don't think we are yet, but we have plenty of runway, so we just focus on getting the products right (and out the door)." (Odilitime)

**Q: All the plugins are still not available yet in the newest version right?**  
A: "We're making tools so we can migrate them plus we're going to add it to the cli, so you can run the migration tool yourself." (Odilitime)

**Q: Will Spartan have a role like defi protocol such as Aave, uniswap?**  
A: "The first Spartan product stuff will be very simple but I think the plan is to iterate on it with community feedback." (Odilitime)

**Q: If X does not respond at all, and Twitter ai16z also hasn't come back yet, what will we do?**  
A: "Backup plans are being discussed ofc. We're going to deploy more comms, on more platforms it sounds like." (Odilitime)

**Q: Why would my agent be getting a new agent ID everytime it starts up?**  
A: "agentid is based on the name, unless the character name is changing this shouldn't happen" (Odilitime)

**Q: How can I create a market data service that stores market data to memories but make those memories accessible in any world and room the agent is added to?**  
A: "Make a service that other plugins can ask for info; use memories or entity/components for storage" (Odilitime)

**Q: How do actions pass data from one action to another?**  
A: "In the future, both actions will reply, so users will get multiple messages" (Odilitime)
