# elizaOS Discord - 2025-06-13

## Overall Discussion Highlights

### ElizaOS V2 Development
- ElizaOS V2 has exited beta and is now at version 1.0.8
- Significant improvements over V1, making development easier
- Team is working on migration tools to port plugins to the newest version
- A CLI tool is being developed to allow users to run migrations themselves using LLMs
- Version 1.0.9 was released by the end of the day, addressing several bugs

### Twitter/X Account Suspension
- The project's Twitter/X account has been suspended, causing community concern
- Team is in talks with X to restore the account
- Developing backup communication plans across multiple platforms
- Community expressed concerns about impact on visibility and price action
- No specific timeline for resolution was provided

### Spartan Product
- An upcoming Spartan product release was mentioned
- Described as a DeFi-focused tool separate from the general-purpose ElizaOS
- First release will be "very simple" with plans to iterate based on community feedback
- Documentation will be created after launch

### Technical Issues
- Database migration errors in v1.0.8 with PostgreSQL schema creation
- Twitter plugin experiencing multiple issues (authentication, inappropriate responses)
- Action handling limitations where data doesn't pass effectively between actions
- Agent deployment methods using Docker were discussed

## Key Questions & Answers

**Q: Do we have any useful or utility agent other than shit-posting?**  
A: "We have over 180 plugins that make it do more than shit posting. You add things like plugin-mcp and the sky is the limit." (Odilitime)

**Q: Do we have any significant updates for Spartan along with v2 release?**  
A: "V2 is out of beta, lots of developers starting to use it. We're already up to 1.0.8 with all the feedback we've gotten. Spartan is about to have his first product release, stay tuned." (Odilitime)

**Q: Will Spartan have a role like defi protocol such as Aave, uniswap?**  
A: "The first spartan product stuff will be very simple but I think the plan is to iterate on it with community feedback." (Odilitime)

**Q: If X does not respond at all, and Twitter ai16z also hasn't come back yet, what will we do?**  
A: "Backup plans are being discussed ofc. We're going to deploy more comms, on more platforms it sounds like." (Odilitime)

**Q: Why am I getting a database migration error with "Failed query: CREATE SCHEMA IF NOT EXISTS $user"?**  
A: "It's a bug in v1.0.8 when creating database tables" (Odilitime)

**Q: How can I create memories accessible in any world and room the agent is added to?**  
A: "You should make a service that other plugins can query for info, using either memories or entity/components for storage" (Odilitime)

**Q: How do actions pass data from one action to another?**  
A: "Currently they don't effectively share data, but this will be fixed in the next release" (sayonara)
