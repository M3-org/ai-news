# elizaOS Discord - 2026-01-06

## Overall Discussion Highlights

### Critical Discord Plugin Issues & Fixes

The Discord integration experienced significant breaking changes in ElizaOS v1.7.0, causing widespread "No server ID found" errors. **Odilitime** identified the root cause as a transition from `serverId` to `messageServerId` in the codebase. The core-devs team merged a critical bug fix (PR #6333), though compatibility issues persisted with Discord 1.3.3, requiring testing across multiple Discord branches and a new Discord release.

**DigitalDiva** encountered the error despite correct intent configurations and admin permissions, with error logs showing the bot couldn't detect server ID, username, or server owner. **Odilitime** created the odi-17 branch with fixes for bootstrap's actions/providers and recommended either downgrading to core v1.6.5 or using the custom branch. **Shaw** suggested debugging with a minimal Discord.js hello world script to isolate permission issues from the Discord developer portal, while **Casino** recommended limiting scope/permissions and working back to desired features.

### Performance Optimizations & Architecture

**Stan** identified multiple latency improvements for the monorepo and prepared comprehensive documentation on cloud fixes addressing TOCTOU (Time-of-check to time-of-use) race conditions using a "deduct-before, reconcile-after" approach. He also worked on runtime initialization optimizations and submitted PRs for Telegram and Discord plugins.

Architecture discussions in core-devs focused on connector gateways and event pumps. The team discussed scaling strategies, noting that voice connections would require higher priority/bandwidth event pumps compared to text, with preprocessing expected to provide significant benefits. The consensus leaned toward simple event pump architecture with plans for multiple daemon instances per service. **Odilitime** recommended reviewing the Jeju cloud branch containing Shaw's preferred Discord bridge implementation.
