# elizaOS Discord - 2025-04-26

## Overall Discussion Highlights

### ElizaOS Ecosystem Development
- **Auto.fun Launch**: The open-source token launching platform (similar to pump.fun) has been operational for less than a week. It's being actively developed by Shaw alongside Eliza v2.
- **ElizaOS Rebranding**: Discussions continue about the rebranding from "ai16z" to "ElizaOS" and potential token migration strategies.
- **Development Philosophy**: The team operates without strict roadmaps, focusing on continuous coding and open-source development, inviting other developers to contribute.
- **Token Security Enhancement**: Auto.fun implementation has improved ai16z security through a liquidity pool funded by buy fees from tokens launched on the platform.

### Technical Issues
- **$QUILL Token Bug**: A significant bug with the $QUILL token allowed someone to drain the liquidity pool by selling tokens at a premium due to incorrect pricing representation. The team acknowledged this was a platform bug (not an exploit) and promised refunds to affected users.
- **Migration Delays**: Several users reported tokens stuck in migration for extended periods (12-20+ hours).
- **Website Performance**: Some users experienced infinite loading issues with the auto.fun website, requiring cache clearing as a potential solution.

### Plugin & Feature Development
- **Livepeer Plugin**: User resethill is seeking assistance with their Livepeer plugin PR (#48) that offers free incentivized inference.
- **3D AI TV Show**: Development continues with implementation of JSON parsing and audio loading with visemes (mouth shapes for speech). Artists need to fix viseme order and schema for each character.
- **Platform Migration**: Discussion about transitioning from Unity to PlayCanvas for the "Clanktank" project, with recommendations to keep backups of the Unity version.

### Governance & Community
- **Token Migration Proposal**: DorianD advocated for a token swap from ai16z to ElizaOS with a 1:1.11 ratio initially, including contacting exchanges and creating new liquidity pools.
- **Discord Moderation**: Recommendations to move dangerous permissions to bots like Wick or Dyno, implementing a tiered moderation system with different permission levels.
- **DegenAI Integration**: Suggestions to either integrate DegenAI into the new ElizaOS token or spin it off as a separate project with its own discord and team.
